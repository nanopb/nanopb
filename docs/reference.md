# Nanopb: API reference

The sections below document the full user-facing API of the nanopb library.

1. [Build options](reference_buildopt.md): Compile-time configuration using `#define` directives to disable features or to set compatibility options.
2. [Generator options](reference_generator.md): Options given to `nanopb_generator`, which affect the generation of C structures from the `.proto` file specification.
3. [pb.h header](reference_pb_h.md): Data types and utility macros used by the whole project.
4. [pb_encode.h header](reference_pb_encode_h.md): Functions used for encoding protobuf messages.
5. [pb_decode.h header](reference_pb_decode_h.md): Functions used for decoding protobuf messages and for memory allocation.
6. [pb_common.h header](reference_pb_common_h.md): Functions used for processing message descriptors and common utility functions.
